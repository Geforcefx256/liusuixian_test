## Context

当前 `ConversationPane` 已经通过 `isRunning`、`canStopRun` 和 `stopPending` 区分空闲、运行中、停止请求中三类状态，但模板仍在运行中并列渲染停止按钮和禁用的发送按钮。输入框和上传按钮在运行中已经禁用，这意味着产品语义本质上是单任务模型，现有双按钮布局只是把状态机表达得不够收敛。

这次变更只涉及 `apps/web/src/components/workbench/ConversationPane.vue` 的交互表达和 `ConversationPane.test.ts` 的断言更新，不调整 `workbenchStore` 的运行、取消、状态收敛逻辑，也不引入新依赖。现有设计系统已经具备图标按钮、danger button、pending spinner 等样式基础，因此可以直接复用内联 SVG 和现有按钮 token。

## Goals / Non-Goals

**Goals:**
- 将发送、停止、停止中三种状态统一到同一个主动作位。
- 用图标化主动作替换当前文字按钮，减少输入区宽度占用和视觉噪音。
- 保持停止动作的危险语义和停止请求中的 pending 反馈。
- 为图标按钮补齐可访问名称、焦点态和禁用态语义。
- 删除输入区底部的固定停止说明文案。

**Non-Goals:**
- 不改变 run cancellation 的后端协议、store 状态机或 `latestStatus` 文案流转。
- 不开放“运行中继续输入并排队发送”的新能力。
- 不重构 workbench 其它按钮、消息卡片或布局结构。
- 不引入第三方图标库或新的样式依赖。

## Decisions

### 1. 使用同一主动作位切换发送与停止

决定保留一个固定的主动作容器，根据 `canStopRun` 和 `stopPending` 切换内部动作，而不是继续并列渲染两个按钮。

- 选择该方案的原因：
  - 与当前单任务模型一致，避免暴露无效发送动作。
  - 视觉上更稳定，用户只需要关注右下角唯一的当前动作。
  - 不需要改动 store 状态机，组件层即可完成收敛。
- 放弃“并列显示停止 + 禁用发送”的原因：
  - 暴露不可执行动作，增加认知噪音。
  - 在窄宽度下占用更多空间。

### 2. 发送与停止都改为图标优先动作

决定把空闲态发送改为上箭头/提交箭头图标，把运行态停止改为 stop 方块图标，并在停止请求中延续 stop 图标语义与 pending 反馈。

- 选择该方案的原因：
  - 图标化按钮更适合聊天输入区的高频主动作位。
  - “发送箭头”和“停止方块”语义清晰，状态切换时识别成本低。
  - 可以保持一个稳定尺寸的按钮外壳，减少布局抖动。
- 放弃“发送图标 + 停止文字”混合方案的原因：
  - 动作位切换时视觉语言不一致。
- 放弃用 `X` 作为停止图标的原因：
  - 更像关闭弹窗，不像停止运行。

### 3. 删除固定停止说明文案，保留系统级状态反馈

决定移除 composer 下方“停止只会取消当前运行...”的固定说明，不在输入区持续占用注意力。

- 选择该方案的原因：
  - 停止请求和运行结果已经由 `latestStatus` 承担反馈。
  - 固定说明不属于高频输入区必须常驻的信息。
- 放弃保留该文案的原因：
  - 会让输入区在运行中显得过度紧张，且与图标化精简方向冲突。

### 4. 用无障碍属性补足 icon-only 控件语义

决定为发送和停止按钮分别提供明确的 `aria-label`，隐藏纯装饰 SVG，并保持现有 button 元素语义与键盘交互。

- 选择该方案的原因：
  - 图标按钮不能依赖视觉猜测，必须显式提供可访问名称。
  - 现有组件已使用原生 `button`，不需要额外键盘处理层。
- 放弃只依赖 `title` 或颜色表达的原因：
  - 读屏和键盘可用性不足，且颜色不是充分语义。

## Risks / Trade-offs

- [图标语义不够清晰] → 使用发送箭头与停止方块这类常见图形，并为按钮提供 `aria-label`。
- [图标按钮点击热区过小] → 保持现有按钮外壳尺寸和最小宽度/高度，不把控件缩成纯图标裸元素。
- [停止中状态被误解为可再次点击] → 在 `stopPending` 时禁用按钮，并保留 pending 指示。
- [测试只覆盖文案，遗漏图标与无障碍语义] → 更新组件测试，断言按钮切换、可见性、disabled 状态和可访问名称。

## Migration Plan

1. 更新 `ConversationPane.vue` 的主动作模板与样式，改为单一动作位切换。
2. 删除停止说明文案，补齐图标与无障碍属性。
3. 更新 `ConversationPane.test.ts`，使断言匹配新的按钮结构和状态表达。
4. 运行相关前端测试，确认空闲、运行中、停止中三态表现一致。
