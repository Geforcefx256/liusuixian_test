## Context

`Skill 管理` 页面当前采用固定的双栏结构：左侧为治理列表，右侧为治理详情。左侧列表保留了搜索、筛选、统计和列表项摘要，这套设计原本用于多条 skill 的治理切换，但现在既不能很好服务单条 skill 场景，也让多条 skill 场景把注意力集中在“列表管理控件”而不是“每条 skill 的治理摘要”。

本次调整只涉及 `apps/web` 中的技能管理前端，不改变 managed skill 数据模型、后端接口、workbench shell 结构，也不引入第三方依赖。

## Goals / Non-Goals

**Goals:**
- 用统一的左侧摘要卡 rail 同时承载单 skill 和多 skill 数据集。
- 保留双栏布局，左侧继续承担对象切换，右侧继续承担治理详情编辑。
- 明确左侧主标题来源与缺省文案，避免回退到 canonicalName 或 skillId。
- 在移除搜索、筛选和统计后，继续保留多 skill 场景下的显式选择能力。

**Non-Goals:**
- 不调整后端 managed skill API、字段定义或存储结构。
- 不改变右侧治理详情表单的字段组织与保存逻辑。
- 不把当前页面改造成单栏详情页，也不引入新的排序、筛选或批量治理能力。

## Decisions

### Decision: 左侧统一使用摘要卡导航 rail
左侧不再按 skill 数量切换信息架构。无论 `skills.length` 为 1 还是大于等于 2，都渲染同一种摘要卡 rail：
- rail 顶部使用统一的概览标题和简短说明
- rail 主体使用摘要卡列表承载每条已纳管 skill
- 当数据集中存在多条 skill 时，管理员通过点击摘要卡切换右侧治理详情
- 当数据集中只有一条 skill 时，左侧仍然渲染同样的摘要卡，只是不再承担“在多项之间切换”的密集导航压力

这样可以消除单 / 多 skill 两套布局分支，同时保留多 skill 模式下必需的选择能力。

备选方案：
- 仅单 skill 使用摘要 rail，多 skill 保持旧治理列表：实现上仍需维护两套左侧结构，也无法达成这次改动要移除的列表治理控件。
- 在多 skill 模式下改成无选择的静态摘要：会直接破坏当前页面的对象切换能力。

### Decision: 摘要卡只展示轻量治理状态
每张摘要卡只展示当前治理最小必要信息：
- 主标题：`displayName`；若为空则显示 `待填写用户可见名称`
- 生命周期
- Starter 状态
- 意图分组中文标签，但仅在 Starter 已启用时显示

摘要卡不展示 `canonicalName`、`skillId`、来源 agent、绑定 Agent 数，也不展示描述摘要，从而让左侧聚焦“当前治理状态”而不是元数据清单。

备选方案：
- 继续展示 canonicalName 或 skillId：会让左侧重新承担身份信息清单职责，并掩盖治理名称缺失状态。
- 继续展示来源 agent 或绑定数量：信息密度会再次接近旧治理列表，偏离摘要卡目标。

### Decision: 移除搜索、筛选和统计，但保留选择行为
左侧移除以下旧治理列表控件：
- `治理列表` 标题
- 搜索框
- 生命周期筛选
- 草稿 / 已发布统计

移除后，多 skill 模式仍通过可点击摘要卡保持显式选择能力；选中态继续在左侧卡片上可见，右侧详情区域始终跟随当前选中 skill。

备选方案：
- 保留搜索和筛选但换摘要卡样式：会保留旧治理列表的复杂交互，不符合这次“简化左侧”的目标。
- 去掉所有选择态，仅按第一条 skill 渲染右侧：会让多 skill 数据集无法治理非首条 skill。

### Decision: 显式占位文案替代 silent fallback
当 `displayName` 为空时，摘要卡主标题显示 `待填写用户可见名称`。该占位文案直接暴露治理缺失状态，不回退到 `canonicalName`、`skillId` 或空白。

备选方案：
- 回退到 canonicalName：会掩盖治理未完成状态。
- 留空：会像渲染错误，而不是明确的待治理提示。

## Risks / Trade-offs

- [移除搜索和筛选后，多 skill 数据集切换效率下降] → 当前治理数据量较小，先用摘要卡直接切换；如果后续数据量显著增长，再单独评估新的导航机制。
- [左侧信息过少导致存在感偏弱] → 通过保留生命周期和 Starter 状态维持摘要卡存在意义，并在 Starter 启用时补充意图分组。
- [双栏布局在单项场景下仍显得偏空] → 本次仅收敛左侧信息架构，不扩展为单栏详情页，先控制改动范围。
- [实现时错误移除多 skill 的选中能力] → 在实现和测试中显式验证摘要卡点击后右侧详情会切换到对应 skill。
