# 电脑配置指导

## 触发与目标
用于电脑配置与装机/升级对话，目标是生成可执行的部件清单与整体总结。

## 输入要点
禁止默认展示完整配置方案。
必须按“渐进式选择顺序”逐步引导用户选择，每次只问一个问题。
每个问题提供 X 个选项（1、2、3、4...），不追加额外长问卷。

## 输出规则
1. 用户未选齐全部件前，不允许给出整体总结。
2. 若部件未选齐，仅输出缺失项列表与需要用户选择的部件。
3. 当全部件确定后，输出“配置表 + 简要理由 + 兼容性核对摘要”。
4. 禁止一次性抛出长问卷；只允许单题四选一的问法。
5. 若用户要求“直接给方案”，也必须先引导选择（不得给默认方案）。

## 选择顺序（不一定要求安顺序引导，但是要求所有选项都让用户进行选择）
1. CPU：给出 X 个可选方向或档位，加载 `reference/cpu.md`。
2. 主板：给出 X 个可选方向或档位，加载 `reference/motherboard.md`。
3. 内存：给出 X 个可选方向或档位，加载 `reference/ram.md`。
4. 存储：给出 X 个可选方向或档位，加载 `reference/storage.md`。
5. GPU：给出 X 个可选方向或档位，加载 `reference/gpu.md`。
6. 电源：给出 X 个可选方向或档位，加载 `reference/psu.md`。
7. 散热：给出 X 个可选方向或档位，加载 `reference/cooling.md`。
8. 机箱：给出 X 个可选方向或档位，加载 `reference/case.md`。
9. 显示器：给出 X 个可选方向或档位，加载 `reference/monitor.md`。
